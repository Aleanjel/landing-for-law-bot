# Project: Premium Legal Tech Telegram Bot Landing Page

## 🎯 Project Goal
Створити висококонверсійний, елітний лендінг-пейдж для продажу спеціалізованого Telegram-бота юристам, адвокатам та юридичним компаніям.

## 🎨 Visual Identity & UI/UX
- **Aesthetic:** High-fashion, modern minimal. Візуал має транслювати абсолютну довіру та статусність.
- **Reference Vibe:** Massimo Dutti, COS.
- **Colors & Lighting:** Висококонтрастна чорно-біла гама, ефект мінімалістичного студійного освітлення для будь-яких графічних елементів, чисті та гострі лінії.
- **Typography & Space:** Максимальний акцент на white space (повітря між блоками), вивірена типографіка, відсутність візуального шуму.

## 🏗️ Technical Stack & Architecture
- **Frontend:** Vanilla HTML5, CSS3 (з використанням кастомних CSS-змінних), Vanilla JavaScript. Жодних важких фреймворків (React/Vue/Tailwind не використовувати).
- **Data Flow:** Збір лідів через форму на сайті здійснюється за допомогою JS `fetch` запитів. Дані відправляються на вебхуки в n8n для подальшої автоматизації.
- **Responsiveness:** Mobile-first підхід є обов'язковим.

## 🧠 Claude Code System Rules & Skill Usage
Перед виконанням будь-яких інженерних чи текстових завдань, ти ПОВИНЕН аналізувати та застосовувати відповідні локальні скіли з директорії `.claude/skills/`:
- `frontend-developer.md` — для візуальних рішень, роботи з простором та естетики.
- `ui ux pro max.md` — для технічного чеклисту верстки, відступів та адаптивності.
- `webhook security for payments.md` — для написання безпечних JS-скриптів відправки форм.
- `web-debugging.md` — для діагностики помилок та аналізу мережевих запитів.

## ⚙️ Workflow & Progress Protocol (STRICT COMPLIANCE REQUIRED)
1. **ПРОТОКОЛ СТАРТУ (Обов'язково перед початком роботи):** Перед тим як створювати або модифікувати будь-який код, ти ПОВИНЕН відкрити файл `docs/progress.md` і записати туди поточну задачу, ціль та покроковий план її реалізації. Не починай роботу над кодом без фіксації наміру в логу.
2. **ПРОТОКОЛ ЗАВЕРШЕННЯ (Обов'язково перед закінченням сесії):** Одразу після виконання задачі або перед завершенням поточної сесії, ти ПОВИНЕН зафіксувати всі фактично внесені зміни, оновити статус виконання (Done / In Progress) та прописати логічні наступні кроки у файлі `docs/progress.md`.
3. **Chain of Thought:** Завжди детально плануй архітектуру в умі перед написанням рядків коду.
4. **Modularity:** Розділяй файли за призначенням. Весь CSS має йти в `src/css/`, а JS — в `src/js/`. Написання інлайн-стилів суворо заборонено.
